Para resolver o problema do loop infinito na seção que apresenta revistas, podemos ajustar o JavaScript e o CSS. Vou sugerir algumas mudanças:

1. JavaScript: A lógica atual está correta em termos de clonar os elementos para criar um efeito de loop contínuo. No entanto, é importante garantir que a largura do container `.scrollerinterno` seja suficiente para acomodar todas as imagens sem que se sobreponham. Além disso, devemos verificar se o loop está sendo reiniciado corretamente.

2. CSS: O CSS parece adequado, mas é necessário garantir que o espaço entre as imagens clonadas e as originais seja suficiente para evitar sobreposições visuais.

Vamos às alterações:

JavaScript

```javascript
document.addEventListener('DOMContentLoaded', (event) => {
const scrollers = document.querySelectorAll('.scroller');

addAnimation();

function addAnimation() {
scrollers.forEach((scroller) => {
scroller.setAttribute('data-animated', true);

const scrollerInner = scroller.querySelector('.scrollerinterno');
const scrollerContent = Array.from(scrollerInner.children);

// Calcula a largura total necessária para evitar sobreposição
let totalWidth = 0;
scrollerContent.forEach((item) => {
totalWidth += item.offsetWidth + parseFloat(getComputedStyle(item).marginRight);
});

scrollerInner.style.width = `${totalWidth 2}px`; // Dobro da largura para acomodar clones

scrollerContent.forEach((item) => {
const duplicatedItem = item.cloneNode(true);
duplicatedItem.setAttribute("aria-hidden", true);
scrollerInner.appendChild(duplicatedItem);
});
});
}
});
```

CSS

Verifique se o espaçamento entre as imagens é suficiente:

```css
.scrollerinterno img {
margin-right: 1rem; / Ajuste conforme necessário */
}
```

Essas alterações devem ajudar a resolver o problema do loop infinito, mantendo a aparência desejada. Lembre-se de testar a seção em diferentes tamanhos de tela para garantir que o comportamento seja consistente.

esqueci de mencionar que ele só não esta aparecendo no iphone

Se a seção de revistas não está sendo exibida corretamente apenas em iPhones, isso sugere que o problema pode estar relacionado à compatibilidade ou a especificidades do navegador Safari, que é o padrão nos dispositivos iOS. Aqui estão algumas sugestões para solucionar o problema:

1. Verificação de Compatibilidade CSS

O Safari tem algumas peculiaridades em relação ao CSS, especialmente com propriedades como `-webkit-mask` e animações CSS. Certifique-se de que todas as propriedades CSS usadas sejam compatíveis com o Safari. Por exemplo, a propriedade `mask` pode não funcionar conforme esperado no Safari. Em vez disso, considere usar `-webkit-mask-image` com gradientes para obter um efeito semelhante.

2. Verificar Problemas de Renderização no Safari

O Safari, em algumas versões, pode ter problemas com a renderização de elementos animados ou com overflow oculto. Verifique se a animação ou as propriedades de overflow estão causando o problema. Em alguns casos, adicionar `transform: translate3d(0,0,0);` aos elementos pode forçar a aceleração por hardware e corrigir problemas de renderização.

3. Testar Eventos JavaScript

Garanta que o evento `DOMContentLoaded` esteja sendo disparado como esperado no Safari. Alguns navegadores têm diferenças sutis no momento em que este evento é acionado. Você pode adicionar um `console.log` para confirmar se o código está sendo executado.

4. Verificar a Visualização de Imagens

Às vezes, a forma como as imagens são carregadas ou exibidas pode variar entre navegadores. Verifique se o caminho das imagens está correto e se não há problemas com o formato ou tamanho das imagens que possam impedir que sejam exibidas no Safari.

5. Testar em Diferentes Dispositivos

Se possível, teste em diferentes modelos de iPhone e versões do iOS para determinar se o problema é específico de um modelo ou versão.

6. Utilizar Ferramentas de Desenvolvimento do Safari

Use as ferramentas de desenvolvimento do Safari para inspecionar a seção problemática e verificar se há erros de console ou problemas de renderização.

7. Considerar Consulta de Mídia Específica para iOS

Em casos extremos, você pode adicionar uma consulta de mídia CSS específica para iOS que ajusta ou simplifica o estilo para esses dispositivos.

```css
@media screen and (max-device-width: 812px) { / iPhone X, por exemplo /
/ Estilos específicos para iPhones /
}
```

Se nenhuma dessas sugestões resolver o problema, talvez seja necessário um diagnóstico mais profundo, o que pode incluir a revisão detalhada do código e testes mais abrangentes em dispositivos iOS.
